üìå When to Use?

  • When recent commits have critical errors
  • Need to go back to a working version
  • Want to start fresh from a previous stable commit
  • Need to rewrite history

üîß Complete Process

1Stay on Main Branch

git checkout main
‚úÖ Expected Output:
Switched to branch 'main'
Your branch is up to date with 'origin/main'.

Check that you're on the correct branch:

git branch
‚úÖ Expected Output:
  dev
* main
  feature-branch
üìù Note: The * (asterisk) indicates the current branch

2Get Code from Previous Commit

git reset --hard <previous_commit_hash>

Example:

git reset --hard f6a3064
‚úÖ Expected Output:
HEAD is now at f6a3064 Previous working commit

What Will Happen:

  • ‚úÖ Main branch HEAD will move to the previous commit
  • ‚úÖ History will revert back to that old commit
  • ‚úÖ Working directory will also be set to that commit's code
  • ‚úÖ No detached HEAD state will occur

3Fix Errors

Edit your code and fix the errors:

  1. Open project files
  2. Make necessary changes
  3. Fix all errors
  4. Test the code

‚úÖ Best Practices

1. Create a backup first:

‚úÖ Expected Output:

2. Verify the commit hash:

Double-check which commit you want to go to

3. Inform your team:

4. Test everything:
